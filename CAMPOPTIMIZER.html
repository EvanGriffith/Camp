How the Optimizer Works

Two views: plain-language summary and full technical detail.

Abstract (plain language)

Each week, 24 cabins rank 13 activities from most to least preferred. The optimizer's job is to assign every cabin to exactly one activity per time slot — 8 slots across Monday through Friday — in a way that makes as many cabins as possible happy.

The key insight is that not all ranks are equal. A cabin's top choices matter far more than their middle ones, and their bottom two choices should be avoided at almost any cost. The optimizer treats a cabin's #1 pick as extremely valuable, their #2 and #3 as very valuable, and the last two as something to avoid. Everything in between scales smoothly.

Two important real-world constraints are baked in: Pool is penalized in morning slots (nobody wants to swim at 9am), and cabins should not repeat an activity across the week — if a cabin already did Archery on Monday, the optimizer will strongly avoid putting them in Archery again later in the week.

Early slip submitters get a tiny tiebreaker advantage — when two cabins are equally well-served by an activity, the one that submitted first wins the spot.

The algorithm runs many random starting points and refines each one through thousands of small improvements, keeping the best result it finds.

Scoring function

Every cabin-activity-slot assignment earns a score. The total schedule score is the sum of all individual assignment scores. The optimizer maximizes this total.

For cabin c assigned to activity a in slot s, where r is the cabin's rank for that activity (1=best, 13=worst):

base_score = 14 - r // 13 pts for rank 1, 1 pt for rank 13 top3_bonus = if r <= 3: (4 - r) * TOP3_MULT // rank1=+9, rank2=+6, rank3=+3 bot2_penalty= if r >= 12: (r - 11) * BOT2_MULT // rank12=-4, rank13=-8 pool_penalty= if activity=Pool AND slot=morning: -POOL_PENALTY repeat_pen = -REPEAT_PENALTY * (times already assigned this activity) tiebreak = (num_cabins - submission_order_index) * 0.001 TOTAL = base_score + top3_bonus - bot2_penalty - pool_penalty - repeat_pen + tiebreak

Default multipliers: TOP3_MULT=3, BOT2_MULT=4, POOL_PENALTY=15, REPEAT_PENALTY=30. All adjustable on the Optimize tab.

Rank score curve

With default weights, the effective score per rank position (before slot/repeat modifiers) is:

Rank 1: +22 pts (base 13 + top3 bonus 9) Rank 2: +17 pts (base 12 + top3 bonus 6) -- note steep drop from rank 1 Rank 3: +14 pts (base 11 + top3 bonus 3) Rank 4: +10 pts (base 10, no bonus/penalty) ...linear... Rank 11: +3 pts Rank 12: -2 pts (base 2 - bot2 penalty 4) Rank 13: -7 pts (base 1 - bot2 penalty 8)

This creates a steep reward at the top, a steep penalty at the bottom, and a smooth linear middle — exactly matching the stated preferences.

Capacity constraints

Each activity has a configurable maximum capacity per slot. The optimizer never exceeds these caps. If an activity is set to 0 for a slot it is unavailable in that slot. Cabins that cannot be placed in any available-capacity activity for a given slot are left unassigned for that slot.

Algorithm: randomized greedy + hill climbing with restarts

Phase 1 — Greedy construction: For each slot, iterate through cabins in a shuffled order. For each cabin, score every available activity (respecting capacity) using the scoring function above plus a small random jitter. Assign the cabin to the highest-scoring available activity. This produces a valid but imperfect schedule.

Phase 2 — Hill climbing: Repeatedly pick two random activity groups within the same slot and swap one cabin between them. If the total schedule score improves, keep the swap. This is a local search that converges to a local optimum.

Phase 3 — Random restarts: Repeat phases 1 and 2 multiple times (default: 8 restarts, each with 6,000 hill-climb iterations). Keep the best schedule found across all restarts. More restarts + more passes = better results but slower runtime.

This approach is a standard iterated local search applied to a combinatorial assignment problem. It is not guaranteed to find the global optimum but reliably finds high-quality solutions for this problem size in under 3 seconds.

Tiebreaker: submission order

The tiebreaker weight (0.001 per position) is intentionally tiny — small enough to never override a genuine preference difference, but large enough to deterministically break ties. The top 3 girl cabin submitters and top 3 guy cabin submitters are given positions 1-3 and 1-3 respectively within their gender group, set on the Setup tab.

Repeat penalty rationale

A repeat penalty of 30 means repeating an activity costs as much as the difference between a rank-1 and rank-4 assignment. The optimizer will therefore accept a repeat only if every non-repeat option scores more than 30 points worse — roughly equivalent to being forced to a rank-4 vs rank-1 choice. In practice, repeats occur only when capacity is very tight across the week.
