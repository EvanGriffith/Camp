How the Optimizer Works

Two views: plain-language summary and full technical detail.

Abstract (plain language)

Each week, cabins rank 13 activities from most to least preferred. The optimizer assigns every active cabin to exactly one activity per time slot — 8 slots across Monday through Friday — in a way that makes as many cabins as possible happy.

Not all ranks are equal. A cabin's top choices matter far more than their middle ones, and their bottom two should be avoided at almost any cost. The scoring curve is steep at both ends and smooth in the middle.

Cabin mixing is encouraged. The optimizer softly penalizes cabins that end up together in too many slots, nudging the schedule toward variety so every cabin gets to meet different groups throughout the week.

Pool is penalized in morning slots. Nobody wants to swim at 9am. Cabins are strongly steered away from Pool during morning assignments.

Repeats are heavily penalized. If a cabin already did Archery on Monday, the optimizer will strongly avoid putting them in Archery again later in the week — but will do so if there is truly no better option.

Early slip submitters get a tiny tiebreaker edge. When two cabins are equally well-served, the one that submitted first wins the spot.

The algorithm runs many random starting points, refines each one through thousands of small swaps, and returns the best schedule found.

Scoring function

For cabin c assigned to activity a in slot s, rank r (1=best, 13=worst):

base_score = 14 - r top3_bonus = if r <= 3: (4 - r) * TOP3_MULT // rank1=+9, rank2=+6, rank3=+3 bot2_penalty = if r >= 12: (r - 11) * BOT2_MULT // rank12=-4, rank13=-8 pool_penalty = if Pool AND morning slot: -POOL_PENALTY repeat_pen = -REPEAT_PENALTY * (times already assigned this activity this week) pair_pen = -PAIR_PENALTY * (times already paired with each co-assigned cabin this week) tiebreak = submission_priority * 0.001 TOTAL = base_score + top3_bonus - bot2_penalty - pool_penalty - repeat_pen - pair_pen + tiebreak

Defaults: TOP3_MULT=3, BOT2_MULT=4, POOL_PENALTY=15, REPEAT_PENALTY=30, PAIR_PENALTY=1.5 (soft).

Pairing penalty

A pairing count matrix tracks how many times each pair of cabins has been assigned to the same activity slot. Each co-assignment beyond the first adds a small penalty (-1.5 per extra pairing). This is intentionally subtle — preference satisfaction always dominates — but over many optimization passes it nudges the algorithm toward more diverse groupings when scores are otherwise close.

Algorithm

Phase 1 — Greedy construction: For each slot, iterate cabins in shuffled order. Score all available activities (respecting capacity) and assign each cabin to its best-scoring option, with small random jitter for diversity.

Phase 2 — Hill climbing: Repeatedly pick two activity groups in the same slot and swap one cabin between them. Keep swaps that improve total score. Runs for N passes per restart.

Phase 3 — Restarts: Repeat phases 1 and 2 multiple times. Return the best schedule found. This is iterated local search — not guaranteed globally optimal but fast and high-quality for this problem size.
